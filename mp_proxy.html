Configuración MercadoPago

Guía para integrar el pago real en Piezauto

1. Credenciales necesarias

Obtené tus credenciales desde el panel de desarrolladores de MercadoPago. Necesitás dos claves:

Public Key — Frontend
Se usa en el browser para inicializar el SDK de MP. No es secreta — puede estar en el HTML.
TEST-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
Access Token — Backend / Proxy
Nunca expongas este token en el frontend. Va solo en tu proxy (Netlify Function, Vercel, Supabase Edge, etc.) como variable de entorno.
TEST-0000000000000000-000000-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx-00000000

Modo Sandbox vs Producción

🧪 Sandbox (TEST)
Las claves que empiezan con TEST- son para pruebas. Podés usar tarjetas de prueba de MP sin cobrar dinero real.
🚀 Producción
Claves sin el prefijo TEST-. Se cobran pagos reales. Activar solo cuando el sitio esté listo para producción.

2. Opción A — Proxy con Netlify Functions / Vercel

Recomendada Sin servidor propio, deploy gratuito

Creás una serverless function que recibe los datos del pedido, llama a la API de MP con tu Access Token, y devuelve el link de pago. El frontend nunca ve el Access Token.

netlify/functions/mp-preference.js

Instalación: npm install mercadopago
Agregá la variable de entorno MP_ACCESS_TOKEN en el panel de Netlify/Vercel. 
// netlify/functions/mp-preference.js
const { MercadoPagoConfig, Preference } = require('mercadopago');

const client = new MercadoPagoConfig({
  accessToken: process.env.MP_ACCESS_TOKEN,
});

exports.handler = async (event) => {
  if (event.httpMethod !== 'POST') {
    return { statusCode: 405, body: 'Method Not Allowed' };
  }

  const { pedidoId, items, payer, backUrls } = JSON.parse(event.body);

  const pref = await new Preference(client).create({
    body: {
      items,
      payer,
      back_urls:        backUrls,
      auto_return:      'approved',
      external_reference: pedidoId,
    },
  });

  return {
    statusCode: 200,
    headers: {
      'Content-Type':                'application/json',
      'Access-Control-Allow-Origin':  '*',
    },
    body: JSON.stringify({
      init_point:         pref.init_point,
      sandbox_init_point: pref.sandbox_init_point,
    }),
  };
};

Para Vercel — api/mp-preference.js

// api/mp-preference.js (Vercel)
const { MercadoPagoConfig, Preference } = require('mercadopago');
const client = new MercadoPagoConfig({ accessToken: process.env.MP_ACCESS_TOKEN });

module.exports = async (req, res) => {
  const { pedidoId, items, payer, backUrls } = req.body;
  const pref = await new Preference(client).create({
    body: { items, payer, back_urls: backUrls, auto_return: 'approved', external_reference: pedidoId },
  });
  res.json({ init_point: pref.init_point, sandbox_init_point: pref.sandbox_init_point });
};

3. Opción B — Supabase Edge Functions

Si ya usás Supabase, podés crear una Edge Function en Deno. No necesitás otro servicio de hosting.

Setup: supabase functions new mp-preference
Luego seteá el secret: supabase secrets set MP_ACCESS_TOKEN=tu-token

supabase/functions/mp-preference/index.ts

// supabase/functions/mp-preference/index.ts
import { serve } from 'https://deno.land/std@0.177.0/http/server.ts';

const MP_ACCESS_TOKEN = Deno.env.get('MP_ACCESS_TOKEN')!;

serve(async (req) => {
  if (req.method === 'OPTIONS') {
    return new Response('ok', {
      headers: {
        'Access-Control-Allow-Origin':  '*',
        'Access-Control-Allow-Headers': 'Content-Type',
      },
    });
  }

  const { pedidoId, items, payer, backUrls } = await req.json();

  const mpRes = await fetch('https://api.mercadopago.com/checkout/preferences', {
    method: 'POST',
    headers: {
      'Content-Type':  'application/json',
      'Authorization': `Bearer ${MP_ACCESS_TOKEN}`,
    },
    body: JSON.stringify({
      items,
      payer,
      back_urls:        backUrls,
      auto_return:      'approved',
      external_reference: pedidoId,
    }),
  });

  const data = await mpRes.json();

  return new Response(JSON.stringify({
    init_point:         data.init_point,
    sandbox_init_point: data.sandbox_init_point,
  }), {
    headers: {
      'Content-Type':               'application/json',
      'Access-Control-Allow-Origin': '*',
    },
  });
});

La URL del endpoint va a ser algo como: https://xxxxxxxxxxxx.supabase.co/functions/v1/mp-preference

4. Configurar en checkout.html

Una vez que tengas tu proxy deployado, editá las dos variables al inicio del script de checkout.html:

// Ejemplo con Netlify Functions:
var MP_PROXY_URL = 'https://tu-sitio.netlify.app/.netlify/functions/mp-preference';
var MP_PUBLIC_KEY = 'TEST-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx';

// Ejemplo con Vercel:
var MP_PROXY_URL = 'https://tu-sitio.vercel.app/api/mp-preference';
var MP_PUBLIC_KEY = 'TEST-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx';

// Ejemplo con Supabase Edge Functions:
var MP_PROXY_URL = 'https://xxxxxxxxxxxx.supabase.co/functions/v1/mp-preference';
var MP_PUBLIC_KEY = 'TEST-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx';

Para ir a producción, reemplazá las claves TEST- por las claves reales de tu cuenta de MP.

5. Checklist para ir a producción

  • 1
    Crear cuenta de MercadoPago de vendedor y activar el modo producción en el panel de developers.
  • 2
    Obtener las claves de producción (sin prefijo TEST-) y reemplazar en MP_PUBLIC_KEY y en la variable de entorno MP_ACCESS_TOKEN del proxy.
  • 3
    Configurar las notificaciones IPN/Webhooks en el panel de MP para que actualicen el estado del pedido en Supabase al recibir el pago.
  • 4
    Implementar el manejo de los query params en seguimiento.html: ?pedido=ID&pago=aprobado|fallido|pendiente para mostrar el estado al usuario.
  • 5
    Implementar la limpieza del carrito (localStorage.removeItem('pz_carrito')) en seguimiento.html cuando el param pago=aprobado.
  • 6
    Probar el flujo completo con las tarjetas de prueba de MP antes de activar producción.
  • 7
    Agregar CORS correcto en el proxy si el frontend y el proxy están en dominios distintos. El header Access-Control-Allow-Origin debe apuntar solo a tu dominio en producción.
